MetroReport (cc_metroreport) (delegates to EventAware, Extractable, Retireable)
Description
MetroReport entity corresponds to the report obtained from Metropolitan Reporting Bureau. MetroReport contains some required data fields for requesting or receiving message from Metropolitan Reporting Bureau (such as loss description, sent date and status). It also has a reference to the generated document. A MetroReport is associated with a single claim.
Entity: MetroReport
Attributes: Editable?, Exportable?, Extendable?, Final?, Keyed?, Loadable?, Sourceable?, Versionable?
Messaging Events: The application will generate these events for this entity:
  • MetroReportRemoved
  • MetroReportAdded
  • MetroReportChanged
Integration programmers can write Event Fired rules to perform actions on these events. Refer to the Integration Guide for the complete list of events for all entities. Remember to configure a new or existing messaging destination in Studio to listen for desired events.
Concrete FK references:
MetroReportWorkflow.MetroReport
Array references:
